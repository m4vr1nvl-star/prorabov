fix: исправление HTTP/2 ошибок на Vercel (ERR_HTTP2_PING_FAILED, ERR_CONNECTION_CLOSED)

## Проблема
Сайт prorabov.ru не загружался полностью на Vercel:
- ERR_HTTP2_PING_FAILED при загрузке main JS bundle
- ERR_CONNECTION_CLOSED для шрифтов и framework JS
- Формы не работали из-за незагруженного JavaScript

## Причина
Конфликт между compress: true в Next.js и встроенным HTTP/2 сжатием Vercel.
Двойное сжатие вызывало обрыв HTTP/2 соединений и таймауты ping frames.

## Решение

### 1. next.config.mjs
- Отключено compress (установлено в false)
- Vercel Edge Network теперь управляет всем сжатием через HTTP/2

### 2. vercel.json
- Удалены дублирующиеся security headers (оставлены только в next.config.mjs)
- Удалены ненужные опции (buildCommand, outputDirectory, framework)
- Улучшены redirects (добавлен www субдомен)

### 3. pages/_document.js
- Добавлен preconnect для wa.me и t.me
- Быстрее устанавливаются соединения для внешних ресурсов
